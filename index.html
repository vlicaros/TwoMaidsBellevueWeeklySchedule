GitHub Setup Guide
One-time setup. After this, your weekly workflow is just: extract → process → upload one file.
1. Create a public GitHub repository
1
Go to github.com → + → New repository

Name: schedule · Visibility: Public · Check "Add a README" · Create.

2
Upload two files from this package

Add file → Upload files → drag in viewer/index.html and an empty schedules.json (content: {}) → Commit.

3
Enable GitHub Pages

Settings → Pages → Source: Deploy from branch → Branch: main / folder: root → Save. Wait ~60 sec.

4
Set your base URL here

Come back to Step 2 of this admin tool and enter your Pages URL so links are generated correctly.

https://vlicaros.github.io/TwoMaidsBellevueWeeklySchedule
2. Configure the viewer (one time)
5
Edit viewer/index.html on GitHub

Open index.html in your repo, click the pencil icon, find the line that says var JSON_URL = params.get('src') || 'SCHEDULES_JSON_URL'; and replace SCHEDULES_JSON_URL with the raw JSON URL below (already done in the viewer file in this package):

https://raw.githubusercontent.com/vlicaros/TwoMaidsBellevueWeeklySchedule/main/schedules.json

Commit the change.
